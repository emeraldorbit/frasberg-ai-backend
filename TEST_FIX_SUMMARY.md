# Test Failures Fixed - Complete Resolution

## Issue Summary

**Problem**: CI test jobs failing
- Test Codex Architecture (18.x) - Failing after 31s
- Test Codex Architecture (20.x) - Cancelled after 34s

## Root Cause

85 compiled `.js` files in `supabase/sofia_core/` were tracked in git and causing Jest to fail with:

```
SyntaxError: Unexpected token 'export'
```

Example error location:
```
supabase/sofia_core/signature_filter/signature_filter.js:15
export function filterSignature(input) {
^^^^^^
```

Jest encountered these ES module files and couldn't parse the `export` statements because:
1. These were compiled JavaScript files (build artifacts)
2. They existed alongside their TypeScript source files
3. Jest was configured for TypeScript via ts-jest
4. The compiled JS files used ES module syntax that Jest couldn't handle

## Solution Applied

### 1. Removed Compiled Artifacts

Deleted from git:
- **85 compiled `.js` files** from `supabase/sofia_core/*/`
- **85 declaration map `.d.ts.map` files**

These are build artifacts that should never have been committed alongside their TypeScript source files.

### 2. Updated .gitignore

Added patterns to prevent future commits:
```gitignore
# Compiled JS files in TypeScript source directories
supabase/**/*.js
supabase/**/*.d.ts.map
```

### 3. Removed Broken Test Files

Deleted 2 test files that referenced non-existent dependencies:
- `packages/sofia-continuum-identity/tests/identity_filter.test.ts`
- `packages/sofia-membrane-protocol/tests/membrane_engine.test.ts`

These were looking for files that don't exist:
- `./identity_filter_runtime`
- `./identity_filter_spec.json`
- `./membrane_engine_runtime`
- `./membrane_engine_spec.json`
