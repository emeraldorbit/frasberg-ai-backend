# Sofia Core v5.1.0 - Quick Start Guide

## 🚀 5-Minute Quick Start

Sofia Core v5.1.0 works **immediately** without any external services!

### 1. Install Dependencies
```bash
pip install -r backend/requirements-v5.1.txt
```

### 2. Start Server
```bash
cd backend/app
python -m uvicorn main:app --reload --port 8000
```

### 3. Test
```bash
# System status
curl http://localhost:8000/

# Interactive docs
open http://localhost:8000/docs
```

## ✅ Done! Running with:
- ✅ SQLite database (automatic)
- ✅ In-memory cache (fallback)
- ✅ Mock LLM (works without API keys)
- ✅ Full authentication

## 🎯 Try It Out

### Register User
```bash
curl -X POST http://localhost:8000/api/v5.1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","email":"test@example.com","password":"secret"}'
```

### Login
```bash
curl -X POST http://localhost:8000/api/v5.1/auth/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=test&password=secret"
```

### Generate (Mock)
```bash
curl -X POST http://localhost:8000/api/v5.1/llm/generate \
  -H "Content-Type: application/json" \
  -d '{"prompt":"Hello Sofia","provider":"openai"}'
```

## 🔧 Optional: Real Integrations

### Enable OpenAI
```bash
cp backend/.env.example backend/.env
# Edit: OPENAI_API_KEY=sk-your-key
```

### Start PostgreSQL
```bash
docker run -d --name sofia-postgres \
  -e POSTGRES_USER=sofia -e POSTGRES_PASSWORD=sofia \
  -e POSTGRES_DB=sofia_core -p 5432:5432 postgres:15
  
# Edit .env: DATABASE_URL=postgresql://sofia:sofia@localhost:5432/sofia_core
```

### Start Redis
```bash
docker run -d --name sofia-redis -p 6379:6379 redis:7-alpine
# Edit .env: REDIS_URL=redis://localhost:6379/0
```

## 🔍 Key Endpoints

- `GET /` - System info
- `GET /docs` - Interactive API docs
- `GET /health` - Health check
- `POST /api/llm/generate` - **Canonical LLM endpoint** (uses `LLM_PROVIDER` / `LLM_API_KEY`)
- `POST /api/v5.1/llm/generate` - Generate completion (v5.1 — provider per request)
- `POST /api/v5.1/auth/register` - Register
- `POST /api/v5.1/auth/token` - Login

## 🤖 Canonical LLM Endpoint

The canonical endpoint is the single sovereign entrypoint for all LLM calls.
Set `LLM_PROVIDER` and `LLM_API_KEY` once; all agents and the voice gateway use it automatically.

### Environment variables

```bash
export LLM_PROVIDER=openai          # or anthropic
export LLM_API_KEY=sk-your-key
export SOFIA_CORE_MODEL=gpt-4.1     # default model
export SOFIA_CORE_TEMPERATURE=0.2
export SOFIA_CORE_MAX_TOKENS=4096
```

### Request

```bash
curl -X POST http://localhost:8000/api/llm/generate \
  -H "Content-Type: application/json" \
  -d '{
    "model": "sofia-core",
    "messages": [
      {"role": "system", "content": "You are Sofia, a helpful AI assistant."},
      {"role": "user", "content": "Hello, Sofia!"}
    ]
  }'
```

### Response

```json
{
  "output": "Hello! I am Sofia. How can I help you today?",
  "usage": {
    "input_tokens": 24,
    "output_tokens": 13
  }
}
```

> **Tip:** Pass `"model": "sofia-core"` (or omit it) to use the defaults from
> `SOFIA_CORE_MODEL`, `SOFIA_CORE_TEMPERATURE`, and `SOFIA_CORE_MAX_TOKENS`.
> Pass any concrete model name (e.g. `"gpt-4o"`) to override.

## 🐳 Docker Compose

```yaml
version: '3.8'
services:
  sofia-core:
    build: .
    ports: ["8000:8000"]
    environment:
      - DATABASE_URL=postgresql://sofia:sofia@postgres:5432/sofia_core
      - REDIS_URL=redis://redis:6379/0
    depends_on: [postgres, redis]
  
  postgres:
    image: postgres:15
    environment:
      POSTGRES_USER: sofia
      POSTGRES_PASSWORD: sofia
      POSTGRES_DB: sofia_core
  
  redis:
    image: redis:7-alpine
```
